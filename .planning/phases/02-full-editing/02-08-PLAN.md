    Server should be running at http://localhost:8000 with the sample model.

    **1. Table CRUD (EDIT-01, EDIT-02, EDIT-03)**
    1. Click "+ Table" in toolbar -- verify new table appears at viewport center with "NewTable" name
    2. Double-click the header of "NewTable" -- verify inline text input appears
    3. Type "UserProfile" and press Enter -- verify header shows "UserProfile (user_profiles)"
    4. Click the table to select it, press Delete -- verify inline confirmation banner appears
    5. Click "Confirm" -- verify table disappears from canvas
    6. Press Ctrl+Z -- verify table reappears
    7. Press Ctrl+Shift+Z -- verify table disappears again

    **2. Column CRUD (EDIT-04, EDIT-05, EDIT-06, EDIT-07)**
    1. Click the "+" icon on the Mailbox table header -- verify new column "new_column" appears
    2. The column name should be in inline edit mode -- type "phone" and press Enter
    3. Click the "phone" column row -- verify property popup opens
    4. Change type to "String(N)", set length to 20, check "Nullable" -- click Apply
    5. Verify the column now shows "String(20)" type and "NN" badge is gone
    6. Click the "id" column -- verify PK checkbox is checked, nullable is unchecked
    7. Click Delete in the popup for "phone" -- verify confirmation banner, click Confirm
    8. Ctrl+Z should restore the deleted column

    **3. Relationship CRUD (REL-01, REL-02, REL-03)**
    1. Drag from the border of "Mailbox" table to "EmailTicket" -- verify dashed blue line during drag
    2. When hovering over EmailTicket, verify it highlights with blue glow
    3. Release on EmailTicket -- verify relationship creation popup appears
    4. Check defaults: 1:N selected, FK column name auto-generated, relationship names auto-generated
    5. Click "Cancel" (the existing relationship already exists)
    6. Click on an existing relationship line between Mailbox and EmailTicket -- verify edit popup opens
    7. Verify back_populates, cascade, lazy fields are populated
    8. Close the popup

    **4. Round-trip Save (PERS-01)**
    1. Make a change (e.g., add a new table "TestTable")
    2. Verify the save button shows a blue dirty dot
    3. Click "Save Model" -- verify green toast "Saved to model.py" appears
    4. Check the terminal/server logs for any save errors
    5. Verify the dirty dot disappears after successful save
    6. Open the model.py file and verify the new table class is present
    7. Verify all original comments and formatting are preserved (TicketStatus docstring, etc.)

    **5. Undo/Redo (UX-02)**
    1. Verify Undo/Redo buttons are disabled when stacks are empty
    2. Make 3 editing operations (add table, rename it, add column)
    3. Click Undo 3 times -- verify each operation reverses in order
    4. Click Redo 3 times -- verify each operation re-applies
    5. Press Ctrl+S -- verify save shortcut works

    **6. Groups (UX-01)**
    1. Click "+ Group" in toolbar -- verify cursor changes to crosshair
    2. Draw a rectangle around Mailbox and EmailTicket tables -- verify rubber band during drag
    3. On release, verify group rectangle appears with "New Group" label
    4. Type a group name "Email System" and press Enter
    5. Right-click the group label -- verify context menu with Rename, Change Color, Delete
    6. Click "Change Color" -- verify color swatch picker, select Green
    7. Drag the group -- verify all embedded tables move together
    8. Drag EmailTicket out of the group boundary -- verify it detaches
    9. Reload the page -- verify group persists with name, color, and members

    **7. Code Preview (UX-04)**
    1. Click "Code Preview" in toolbar -- verify panel slides open on the right
    2. Verify Python code is displayed with syntax highlighting
    3. Click a table on canvas -- verify preview auto-scrolls to that table's class
    4. Make an edit (e.g., rename a column) -- verify preview updates after ~500ms
    5. Drag the resize handle left/right -- verify panel resizes
    6. Click X to close -- verify panel closes and tab appears on right edge
    7. Click the tab -- verify panel reopens

    **8. Collapse Chevron (Phase 1 remap)**
    1. Verify chevron icon (V) visible on the right side of table headers
    2. Click the chevron on Mailbox -- verify table collapses (shows only PK/FK columns)
    3. Click again -- verify table expands
    4. Double-click the header text -- verify it enters inline rename mode (NOT collapse)
